CANON × Hospital Operations

A technical reference atlas for systems-minded readers in clinical operations, informatics, and applied systems analysis. CANON is a constrained state-evolution framework. This atlas instantiates it for hospital bed management and LOS containment — specifying the governing equation's bind points (F, K, and state variables) in explicit operational terms.

x(t+1) = ΠK(F(x(t)))

F = unconstrained ADT flow, staffing draws, acuity activation.   K = the feasibility constraints that must hold.   Everything else is derived from or subordinate to this.

§1 — The Core Distinction

Latent state, observed trace, rendered view.

CANON makes a structural separation that operational systems routinely collapse. The formal consequence: decisions made from the KPI layer alone incur a recoverable cost — projection loss LP — visible as coordination burden on the receiving shift.

Three layers

LayerCANON termHospital example
What the system is doing X̃ (latent state) Float pool state, true staffing load, acuity mismatch, EVS rhythm
What measurement captures Y = PK(X̃) Census metrics, KPI values, shift summary reports
What the analyst sees Mview Dashboard charts, leadership rounding printouts
Structural consequence: Y is a projection of X̃, not a substitute for it. Information destroyed in the projection = LP. The next shift pays that cost as reconstruction burden.

The contain / uncertain / losing judgment

This classification — used operationally by bed coordinators — is the clinician-readable instantiation of Δc*. It integrates staffing margin, path history, accessible futures, and boundary proximity simultaneously. KPI dashboards do not.

The LOS application spec (See domain/los_mapping.json and spec/CANON_SYSTEM_v3.9.53.json) encodes this judgment as LOS.contain, LOS.uncertain, and LOS.losing. The theory formalized a judgment that was already operational.

F_locopt
Local metric improves; system coherence degrades. Problem transferred, not resolved.
F_overctl
Regulation stabilizes now; accessible futures collapse. No moves left for hour-6 surge.
F_dimred
Dashboard green; latent state deteriorating. LP is high. Next shift inherits the gap.
F_pathlock
Identical current margin; different trajectory. Three prior shifts locked the futures.

§3 — The Math

The extended collapse margin.

Δc* is the distance to the collapse boundary, incorporating history, cross-plane coherence, accessible futures, and projection loss. This is the formal structure underlying the contain / uncertain / losing operational judgment.

Complete formula

Δc* = w₀ + w₁·Ω_V + w₂·Π + w₃·C_x + w₄·A_s − w₅·Φ − w₆·L_P − w₇·‖H‖ − w₈·λ_growth − w₉·σ_penalty − w_HD·‖H‖·Φ (path-dissipation coupling — default off) − w_LS·L_P·σ (projection-boundary coupling — default off)

All weights wi ≥ 0. Setting w_HD = 0 and w_LS = 0 recovers the separable baseline exactly (v3.9.53 compatibility). Interaction terms activate when path-lock and dissipation compound.

LOS.stress is the operational inverse of Δc*. A unit can show normal census and high stress if H and Φ are elevated.

§4 — Failure Channels

Four structurally distinct failure modes.

Each has a distinct signature in the state variables and is distinguishable in latent space. These are the failure modes that operational language typically collapses into "the shift was hard."

F_locopt
Local optimization · global degradation

One unit clears its boarders to hit its LOS target, pushing complex patients to an adjacent unit. The system looks better in aggregate KPIs while actual risk has concentrated downstream.

Clinical vignette: Med-Surg A discharges three borderline patients to meet census targets. All three require readmission within 6 hours. Unit B receives two complex transfers without capacity. LOS.margin on A looks green. LOS.losing activates on B.
Signature: d(local Δc)/dt > 0 while d(global C_x)/dt < 0
margin improving in one unit; boarders rising adjacently; C_load degrading
F_overctl
Over-regulation · future options consumed

The bed coordinator bends rules, calls exceptions, authorizes overtime to hold the shift. It works — until As is exhausted. No moves remain for the surge at hour 6.

Clinical vignette: Coordinator calls 4 exceptions by hour 3, draws float pool to zero. Census looks controlled. Hour 6 surge arrives. There is nothing left to pull. Unit crosses into terminal regime.
Signature: Π up, short-term Δc* up, Ω_V down, A_s down
exceptions_count high; staffing_margin briefly stable; float_burn_rate critical
F_dimred
Dimensional reduction · decisions from compressed view

Decisions are driven from Y — the KPI projection — while LP is high. The dashboard looks stable while the latent state is deteriorating. The next shift inherits a state the view didn't show.

Clinical vignette: 0700 huddle: census 94%, LOS within target, staffing "adequate." Not visible: float_burn_rate = 0.82, interrupts_per_rn_hour = 4.2, EVS median = 47 min. Night charge nurse knows. The dashboard does not.
Signature: L_P up while policy depends primarily on Y = P_K(X̃)
kpi_value stable; LOS.stress rising; float_burn_rate high
F_pathlock
Historical grooves · locked future transitions

Three prior high-census shifts burned the float pool. H is high. Today's ADT is moderate. As is near zero. Identical current margin, no recovery moves — not from today's inputs but from the trajectory preceding them.

Clinical vignette: Mon–Wed burned the float. Thursday ADT is light. Staffing_margin looks fine. But float_burn_rate = 1.4, carrying three shifts of H. A mild complexity spike at 1400 activates F_pathlock. No recovery path exists.
Signature: H up, A_s down
float_burn_rate high; LOS.losing = true despite moderate T
Diagnostic heuristic: Metric improves while cross-plane function degrades → F_locopt or F_dimred. Order improves but flexibility collapses → F_overctl. Present looks adequate but no viable moves remain → F_pathlock.

Regulate ≠ Constrain. Regulate applies corrective pressure within K — governance preference. Constrain enforces the boundary — the projection ΠK that removes inadmissible states. Sustained over-Regulation reduces As. Constrain is terminal. Conflating them is the structural source of F_overctl.

Observation chain

PK is derived from the same K that governs state evolution. Collapse is decided in latent or boundary space — never in view space. Rendered views do not modify X̃.

§7 — Execution Semantics

Reference execution order and interpreter roadmap.

Step order for a conformant implementation against the declared LOS spec. Numeric declarations required for reproducibility per v3.9.53 closure requirements.

Execution step order

  1. Ingest LOS event stream for [period_start, period_end]
  2. Build Xc: {Ω_V, Π, T, Γ} from LOS variables per §2 formulas
  3. Build X̃: add H (float_burn_rate), θ (shift_start phase), ω (EVS cadence)
  4. Apply Enable → Express → Regulate (factored latent update)
  5. Apply Constrain: project onto K — check all feasibility constraints
  6. Finalize X̃(t+1)
  7. Compute derived: Φ = T·Γ, Δc = f(Ω_V,Π,Φ), Δc* per weighted formula
  8. Classify: contain / uncertain / losing from Δc* thresholds
  9. Compute Y = PK(X̃): KPI projection layer
  10. Compute LP: information gap between X̃ and Y
  11. Generate trace Z from event log via Probe(Y, S, W)
  12. Apply Filter(Z) → Render → M_view
  13. Emit: LOS.stress, LOS.margin, float_burn_rate(t+1), ΣK regime

Numeric declarations (v3.9.53 required)

h_norm_family: absolute_value (scalar H) Q_family: jacobian_induced_tangent_metric f_star_mode: off (interactions default off) lambda_H: 0.7 (30% decay per shift) eps_Q: 1e-6 (regularization floor)

Validation preflight

  • rn_required and rn_scheduled ≥ 0, non-null
  • staffable_beds ≥ 0
  • period_start < period_end
  • float_burn_rate ≥ 0 (h_norm = |H|, must be nonneg)
  • Qactive positive definite on {Ω_V, Π} subspace
  • All wi ≥ 0 in Δc* formula
